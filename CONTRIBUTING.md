Contributing to the Climate & Phenology Journal

Thank you for your interest in contributing!
This project is a collaborative, public‑good effort to build a long‑term climate and phenology record for any region. We welcome contributions from developers, researchers, designers, ecologists, emergency services, educators, and citizen scientists.

This guide explains how to get involved.

Ways to Contribute

1. Improve the App
  Build features (observation entry, offline mode, dashboards)
  Improve UI/UX
  Add region‑specific phenology indicators
  Enhance accessibility

2. Improve the Data Model
  Propose new fields
  Suggest region‑specific variables
  Help refine phenology stages
  Add species lists for different climates

3. Documentation
  Improve README or this CONTRIBUTING guide
  Add examples or tutorials
  Create diagrams or flowcharts

4. Research & Domain Knowledge
  Provide ecological expertise
  Suggest wildfire‑awareness indicators
  Share phenology knowledge from your region

5. Community Engagement
  Share the project with local groups
  Help test early versions
  Provide feedback from field use

How to Get Started

Step 1 — Check the Issues
  Look at the Issues tab to find:
  open tasks
  feature requests
  bugs
  discussions
  If something interests you, comment to say you’re working on it.

Step 2 — Fork the Repository
  Click Fork to create your own copy of the project.

Step 3 — Make Your Changes
  Follow the project structure and coding style.
  If you’re unsure, ask in an Issue — we’re friendly.

Step 4 — Submit a Pull Request
  When your work is ready:
  open a Pull Request
  describe what you changed
  link to any related Issues
  We’ll review it together.

Code of Conduct
We aim to create a welcoming, respectful, and collaborative environment.

Please:
  be kind
  be constructive
  respect diverse expertise
  assume good intentions

Everyone is here because they care about the land.

Questions or Ideas?

Open an Issue anytime.
You don’t need permission to start a conversation — your perspective matters.
